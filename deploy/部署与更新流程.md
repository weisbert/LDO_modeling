# LDO 建模器 —— 黄区 → 红区 部署与更新（实战流程）

本文记录实际跑通的整套流程与踩过的坑。参考版（更细）见同目录 `操作手册_OPERATIONS.md`。

---

## 0. 环境与约定

| 区 | 系统 | 网络 | Shell | 职责 |
|---|---|---|---|---|
| **黄区** | Windows | 有网 | PowerShell | `git pull` + 打包（下轮子、审计、打 tar） |
| **红区** | Linux（CentOS7 类，glibc 2.17，python3.11） | 气隙无网 | **tcsh/csh** | 安装、运行 GUI |

- **安装位置**：在红区自建一个文件夹（如 `~/workarea/LDO_modeling`），**整套就装在它下面**（扁平：`.venv app wheels results model` 直接在里面，无 `install/` 子层）。
- **PREFIX = 这个文件夹本身**，用 shell 内置的 **`$PWD`**；**不要自己写 `ROOT=...`**（EDA 环境常已占用 `$ROOT`，会冲突；而且 tcsh 里 `VAR=val` 语法本来就不对）。
- GUI 是**纯解析**对比，红区**不需要 ngspice/Spectre**，只要 numpy/scipy/matplotlib/PyQt5。

---

## 1. 黄区：打包（PowerShell）

```powershell
cd D:\path\to\LDO_modeling
git pull origin main                       # 先拉最新代码

# 完整包（首次部署 / 依赖变更时）
.\deploy\package.ps1                        # 产物：dist\ldo_modeler_full.tar.gz (+ .sha256)

# 增量包（只改了代码、依赖没变）
.\deploy\package.ps1 -Mode incremental      # 产物：dist\ldo_modeler_incremental.tar.gz (+ .sha256)
```

`package.ps1` 自动：找 Python 3.11 → （full）跨平台下红区轮子 + 审计 glibc ≤ 2.17 + 冻结 lock → 写 MANIFEST/sha256（**LF 行尾、正斜杠路径**）→ 打 tar。
若增量包中止报 `requirements changed` → 依赖确实变了，改打完整包。

---

## 2. 传输到红区

用审批介质，把对应的 `*.tar.gz` + `*.sha256` 拷到红区的 `LDO_modeling` 文件夹。
**只需一次**：把两个启动器也放进去 —— `deploy/run_gui`、`deploy/update`
（黄区 `git pull` 后在 `deploy/` 里取；或 raw 下载
`https://raw.githubusercontent.com/weisbert/LDO_modeling/main/deploy/update`）。

---

## 3. 红区：首次部署（完整包）

```tcsh
cd /data/.../workarea/LDO_modeling                     # 安装目录 = $PWD
sha256sum -c ldo_modeler_full.tar.gz.sha256            # 落地校验（新包 LF，直接校）
mkdir -p bundle && tar xzf ldo_modeler_full.tar.gz -C bundle
bash bundle/bootstrap.sh "$PWD"                         # PREFIX = 这个文件夹本身
```

`bootstrap.sh` 自动：逐文件校验 → 铺 `app/`+`wheels/` → 建 `.venv` → **离线** `pip --no-index` → **冒烟测试（已内置自带 Qt 隔离）** → 生成 `run_gui`/`update` 启动器。
看到 `GUI selftest PASS` 即成功。装完 `bundle/` 和 tar 可删：`rm -rf bundle *.tar.gz*`。

启动 GUI：
```tcsh
bash run_gui            # 或：chmod +x run_gui 后用 ./run_gui ；需要显示（X11 转发 / VNC）
```

---

## 4. 红区：增量更新（以后日常，一条命令）

```tcsh
cd /data/.../workarea/LDO_modeling
# 把新的 ldo_modeler_incremental.tar.gz 传进这个文件夹
bash update            # 或：chmod +x update 后用 ./update
```

`update` 自动：校验 → 解包到临时目录 → 跑 `update.sh`（换 `app/`，**保留 `.venv`/`wheels`/`results`/`model`**，Qt 隔离冒烟）→ 清理临时目录。
以后每次更新就两步：**传新的增量 tar → `bash update`**。

---

## 5. 关键坑 & 对策（本次实际踩过的）

| 现象 | 原因 | 对策 |
|---|---|---|
| `./update`/`./run_gui` 报 `Permission denied` | 上传（scp/拷贝）丢了执行位 | `bash update`（免 chmod）；或 `chmod +x update` 后 `./update`（一次性） |
| `run_gui`/`update` 不在 `LDO_modeling/` 下（在 `app/deploy/` 里） | 手动安装跳过了 bootstrap 把启动器拷到根目录这一步 | `cp app/deploy/run_gui app/deploy/update . && chmod +x run_gui update`（新版 `update.sh` 已会每次自动刷到根目录） |
| `sha256sum -c` 报 `No such file`、文件名带 `\r` | 旧 Windows 包的边车是 CRLF | `sed 's/\r$//' *.sha256 \| sha256sum -c`（新包已 LF，无需） |
| bootstrap 校验**全部 MISSING**（`app\cadence\...` 反斜杠） | 旧包 MANIFEST 用了 `\` 路径键 | 用新包（已 `as_posix` 出正斜杠）；旧包走第 6 节手动铺 |
| 导入报 `symbol _ZdaPvm, version Qt_5 not defined in file libQt5Core.so.5` | 机器上 Cadence/系统 Qt（如 `/software/public/qt/5.15.3_xcb/lib`）盖住了自带 Qt 5.15.2 | 用 `run_gui`（已内置把自带 Qt 前置）；bootstrap/update 的冒烟也已自动前置 |
| `ROOT=...` 报 `Command not found` / `$ROOT` 报 `Undefined variable` | 红区是 **tcsh**，不是 bash | 别用 `VAR=val`；用 `$PWD`、`env VAR=val cmd`，或把 bash 语法交给 `bash 脚本` 跑 |
| `mkdir: cannot create '/opt/...'：Permission denied` | 共享机对 `/opt` 没写权限 | PREFIX 用你自建文件夹（`bash bundle/bootstrap.sh "$PWD"`） |
| `python3.11 not found` | 解释器不在 PATH | `PYTHON=/abs/path/python3.11 bash bundle/bootstrap.sh "$PWD"` |
| 纯计算节点冒烟因缺 Qt `.so` 失败 | 该节点无图形库 | `SMOKE_REQUIRE_QT=0 bash bundle/bootstrap.sh "$PWD"`（装包与 numpy/scipy 仍验证） |

## 6. 兜底：手动铺（仅当拿到的是**旧包**、bootstrap 校验门坏时）

整包 `sha256` 已 OK 即说明包是好的，可跳过坏掉的逐文件门、照 bootstrap 步骤手动装（全是外部命令，tcsh 直接能跑、零变量）：

```tcsh
cd /data/.../workarea/LDO_modeling
cp -r bundle/app bundle/wheels .
cp bundle/requirements.lock bundle/update.sh .
cp bundle/MANIFEST.json ./MANIFEST.deployed.json
mkdir -p results model
ln -sfn ../results app/results
ln -sfn ../model   app/model
python3.11 -m venv .venv
.venv/bin/pip install --no-index --find-links=wheels -r requirements.lock
env LD_LIBRARY_PATH=$PWD/.venv/lib/python3.11/site-packages/PyQt5/Qt5/lib QT_QPA_PLATFORM=offscreen .venv/bin/python app/gui/ldo_modeler.py --selftest --require-qt
```

（新包用不到这节；新 bootstrap/update 已自带 Qt 隔离、LF、正斜杠路径，正常一把过。）

## 7. 命令速查

```
# —— 黄区（Windows，有网）——
git pull origin main
.\deploy\package.ps1                       # 完整包
.\deploy\package.ps1 -Mode incremental     # 增量包

# —— 红区（Linux / tcsh，气隙；先 cd 到你自建的文件夹）——
# 首次：
sha256sum -c ldo_modeler_full.tar.gz.sha256
mkdir -p bundle && tar xzf ldo_modeler_full.tar.gz -C bundle
bash bundle/bootstrap.sh "$PWD"
# 更新：
bash update                                # 传入新的 incremental tar 后
# 启动：
bash run_gui                               # 需显示（X11/VNC）
```
