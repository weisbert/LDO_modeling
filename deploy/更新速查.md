# 更新速查

详细说明见 `部署与更新流程.md`，这里只有命令。

## 黄区 Windows：代码更新 + 打包

```powershell
git pull origin main
.\deploy\package.ps1 -Mode incremental    # 增量包（日常，只改了代码）
.\deploy\package.ps1                      # 完整包（首次 / 依赖变了）
```

产物在 `dist\`：`ldo_modeler_*.tar.gz` + `.sha256`，两个一起拷去红区。

## 红区 Linux：更新（增量 / 全量，一条命令）

把任意包（`ldo_modeler_incremental.tar.gz` 或 `ldo_modeler_full.tar.gz`，连同 `.sha256`）传进
安装文件夹，然后**一条命令** —— `apply` 自动读包里的 mode，增量走 `update.sh`、全量走
`bootstrap.sh`，原地更新，`.venv/results/model` 该留的都留：

```tcsh
cd /data/RFIC3/Hi1108V100_Pilot_C1Xplus/w84368867/workarea/LDO_modeling
bash apply                              # 自动选目录里最新的那个 tar
bash apply ldo_modeler_full.tar.gz      # 或显式指定
```

`bash update`（仅增量）仍可用，但 `apply` 两种都管，日常推荐它。

## 首次让 apply 就位（仅一次，apply 还没装到安装根时）

`apply` 随包发布到 `app/deploy/apply`，但安装根上还没有它时，先把它从包里取出来再跑一次
（之后它会自动常驻安装根，以后直接 `bash apply`）：

```tcsh
cd /data/RFIC3/Hi1108V100_Pilot_C1Xplus/w84368867/workarea/LDO_modeling
tar xzOf ldo_modeler_full.tar.gz app/deploy/apply > apply && chmod +x apply
bash apply                              # 这次会跑全量 bootstrap，并把 apply 常驻安装根
```

## 红区 Linux：从零首次部署（裸机，连 .venv 都没有）

```tcsh
cd /data/RFIC3/Hi1108V100_Pilot_C1Xplus/w84368867/workarea/LDO_modeling
sha256sum -c ldo_modeler_full.tar.gz.sha256
mkdir -p bundle && tar xzf ldo_modeler_full.tar.gz -C bundle
bash bundle/bootstrap.sh "$PWD"
```

## 红区 Linux：启动 GUI

```tcsh
bash run_gui    # 需要显示（X11/VNC）
```
