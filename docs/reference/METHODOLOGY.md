# METHODOLOGY ‚ÄĒ the modeling method + acceptance gates (what works AND what was refuted)

> Durable archive. Each method carries a verdict: **WORKS / REJECTED / SUPERSEDED / OPEN**.
> Refuted methods are kept deliberately ‚ÄĒ they are as valuable as the working ones (they stop us re-trying dead ends).

## Core architecture

- **Linear 2-port LTI model = Zout(s) + PSRR(s) + shaped Norton noise + discrete spurs + a SEPARATE nonlinear DC/dropout table** ‚ÄĒ **WORKS (conditional).** The FORM is physically correct for disturbances ABOVE the loop UGB (loop open ‚Üí vout = the open-loop passive output network). FORM holds within the declared envelope; the historical VALIDATION was in-sample/block-level only (see Validation below).
- **The irreducible kernel = ONE phase-accurate, OP-parameterized Zout(s)** ‚ÄĒ **WORKS (the central insight).** All four observables are the SAME Zout driven by different sources: PSRR = Y_couple¬∑Zout¬∑vin; noise Sv = |Zout|¬≤¬∑S_in; transient = IFT(Zout¬∑i_load) (carries NO new info beyond Zout in the linear regime); spur = Zout¬∑i_spur. Only DC/dropout is a genuinely separate nonlinear object. ‚áí a Zout error is COMMON-MODE across all four.
- **One emitted instance, layered (NOT split per use-case)** ‚ÄĒ **ADOPTED.** Deployment needs ONE `.va`/`.lib` (a system PSS/HB run presents Zout+PSRR+noise+spur simultaneously on one node ‚ÄĒ separate fits would let them disagree about the one physical Zout = unphysical). Characterization is already per-block. Form = one shared phase-first identifiability-gated kernel + per-use shells + a use-case-resolved scorecard.
- **Per-block swappable blocks + data-driven topology selectors** ‚ÄĒ **WORKS.** 4+2+2 blocks compose vs a 4√ó2√ó2 monolithic cross-product. Confirmed by an independent survey to be the field's recommended composite.
- **PVT = route A: characterize each PVT cell separately ‚Üí fit separately ‚Üí emit `.lib` SECTIONS** (tt_25c_1v05, ss_125c_0v95‚Ä¶), Cadence corner setup picks the section. **ADOPTED** (deliberately avoids a new overfitting surface). Cross-PVT interpolation **REJECTED**; continuous PTAT-ish i(T) is the only exception.

## Zout block

- **Multi-start bounded TRF + R_pl damping resistor + optional 2nd parallel R-L branch** (engaged only when it beats 1-branch by >40%) ‚ÄĒ **WORKS.** R_pl‚Üí‚ąě = resonant peak; finite R_pl = resistive plateau (one DOF generalizes peaked‚ÜĒplateau). Branch B closes V3 (3.45‚Üí0.78 dB).
- **Higher-order (L‚ÄĖR) ladder in series** (Ra + N√ó(L‚ÄĖR) + Cout/ESR; magnitude-shape gate) ‚ÄĒ **WORKS** for a rising-then-plateau resistive shelf (the real WuR rails). Productionized through the in-situ pipeline, byte-identical when the `extra` list is absent.
- **Single inductor (La‚ÄĖRpl)+Cout to model a rising shelf** ‚ÄĒ **FAILED** (can't make rising-then-plateau ‚Üí mislocates the corner ~70√ó, to 447 kHz).
- **Cout/ESR auto-extract from the capacitive band** (‚ą†Z<‚ąí45¬į, C=‚ąí1/(ŌČ¬∑Im Z)) ‚ÄĒ **WORKS** within a few % over 100 pF‚Äď10 nF, EXCEPT high-ESR/cap-less parts (v1 381 vs 1000 pF; 30 ő© ESR floors the HF tail). Joint-LS Cout/ESR for those ‚ÄĒ **REJECTED** (underdetermined when ESR‚Čęoutput-R, the cap is near-invisible ‚Üí diverges; reverted to median/keep-best).
- **Unbounded LM + argmax-of-flat init** ‚ÄĒ **REJECTED** (diverged: cout10n R_a=32 Mő©; V1 92 dB spurious peak; v2 Cout ‚Üí 1e269 F).
- **Passive-by-construction RLC** ‚ÄĒ **WORKS** (the #1 HB-convergence lever). It CANNOT reproduce a genuinely non-passive GT (v3 Re(Z)<0, ~‚ąí0.25 ő© floor); that residual is a documented FLOOR, not a bug. Non-passive controlled-source realization **REJECTED** (HB-stability risk).
- **Branch-A regulation must be the STIFF R_a resistor (`V(nA,vrg) <+ Ra*I`), NOT a saturating "DC current compliance"** ‚ÄĒ **REJECTED 2026-07-01** (`max(-Icomp,min(Icomp,V/Ra))` / `Icomp*tanh`). The clamp's knee is a **voltage** `Icomp¬∑Ra`; with the near-zero fitted Ra (~0.02‚Äď0.1 ő©) that's **~1‚Äď5 mV**, so a few mV off vreg the regulation becomes a **zero-conductance current source** ‚Üí the output DC pin is lost ‚Üí **FF-corner runaway (Spectre: 2.31e6 V) / non-convergence** (TT/SS only survive by staying in the ¬ĪmV sliver). The resistor holds 1/Ra everywhere and is bit-identical in Zout/PSRR/noise + validated large-signal. LESSON: a current-limit on a near-ideal source is only well-conditioned if the knee VOLTAGE ‚Čę any excursion the DC solve traverses; a real dropout/current-limit is stage-2b and must be a one-sided, conductance-preserving soft limit. (The "unbounded DC fight" it targeted is handled by the PSRR supply auto-track instead. `harness/test_pvt_robust_emit.py` locks this.)
- **Zout shelf fit = sign-agnostic SHAPE gate, magnitude-only, cap-open** ‚ÄĒ **WORKS.** The old "resonance mislocated / Q2000" diagnosis was WRONG (a metric artifact on a loop-active negative-real shelf). importmp passivity-normalizes Zout to Re‚Č•0, so the gate must be shape-only (Q>3 + ipk‚Č•0.6N + plateau>0.6peak), not Re-sign-dependent.

## PSRR block

- **Identify i_c = H_psrr/Zout by Sanathanan‚ÄďKoerner rational fit (freq-scaled, relative-weighted); realize as a bank of signed first-order real-pole sections** ‚ÄĒ **WORKS.** The earlier claim "PSRR doesn't factor as i_c¬∑Zout" was WRONG/REFUTED ‚ÄĒ it was a fit-method failure (shelf insufficient order; naive fit won't converge over 60 dB), not structural.
- **One signed complex-conjugate 2nd-order section, AAA-initialized + least_squares-polished on the realizable form** ‚ÄĒ **WORKS** for non-min-phase parts (v4 mag to 0.04‚Äď0.06 dB; 25¬į residual phase remains). N2=1 is the sweet spot. Keep a 1-section shelf if min-phase fits (no regression on 9 variants); the 3-real bank polish must be UNGATED (was wrongly `if CFT>0`).
- **Raw AAA poles dumped into the model** ‚ÄĒ **REJECTED** (over-fits: 3‚Äď6 spurious pairs, Q~1700). AAA is an INITIALIZER only.
- **Ranking a real PSRR notch fit by analytic residual** ‚ÄĒ **REJECTED** (low analytic residual but huge realized phase error from fragile pole-zero cancellation). Always realize + score, never rank by analytic residual.
- **PSRR is NOT non-minimum-phase by default** ‚ÄĒ the Hilbert "non-min-phase" reading was an artifact of sparse silicon AC points.
- **AAA complex-section initializer on sparse silicon AC** ‚ÄĒ **FAILED / OPEN** ‚ÄĒ the remaining WuR **pll PSRR blocker** (resid 3.16 > shelf 1.06, ~47-pt sweep, phase RMS ~51¬į). Needs a robust complex-section initializer for sparse real AC.
- **Two separated PSRR notches in a single-loop integrated LDO** ‚ÄĒ **REJECTED (physics).** Two feedforward paths merge into one effective notch; an on-chip LC trap at 10s of MHz needs unphysical ¬ĶH. The single-complex-pair block matches single-loop physics; exceeding it needs a cascade/multi-loop topology.

## Noise block

- **Decoupled Norton-@vout: In = Sv_meas/|Zout_model|, In¬≤ = white + 6 Lorentzians fit in the LOG domain jointly over 3 corners (shared corner freqs, per-corner amplitudes); realize as white-R + 6 R‚ÄĖC VCCS sections** ‚ÄĒ **WORKS.**
- **log-AMPLITUDE noise fit** ‚ÄĒ **WORKS**; replaced linear-LS-in-power (LF flicker inflated the white floor 5‚Äď30√ó; real flicker af~0.85 sub-1/f).
- **CAVEAT ‚ÄĒ "decoupled" is a MISLABEL, not a defect.** It's an algebraic round-trip (In=Sv/|Z| ‚Üí Sv=In¬∑|Z|) that decouples SYNTHESIS, not PHYSICS ‚ÄĒ a Zout error leaks identically into noise. DO NOT "fix" the round-trip (the physics is correct); relabel it and score Sv vs GT end-to-end.
- **Series-voltage noise in branch A riding the Cout divider** ‚ÄĒ **REJECTED** (base-specific + coupled to Zout synthesis; adding branch B for V3 perturbed noise 5.6‚Üí21.8 dB). EXCEPT: a **HYBRID series-voltage bank WORKS for loop-shaped rails** (Norton can't hold In=Sv/|Zout| for the pll/vco ~11 dB rails; synthetic DUTs stay Norton).
- **`flicker_noise()` / `noise_table()` for the 1/f tail** ‚ÄĒ **REJECTED** (no byte-matching ngspice analog ‚Üí forfeits the 9e-6 cross-engine lock; the free Lorentzian bank is equal-or-better).
- **Spur sampling rule** ‚ÄĒ **LOCKED.** `.noise`/AC sweeps MUST sample the spur center + very-short-step points (0.05‚Ä¶4√óHWHM, Q-aware) around it, else the narrow spur is stepped over (54√ó miss measured).

## Spur block

- **Deterministic vout current tones `Isp 0 vout`, I_k = vout_amp_k/|Zout_model(f_k)|, characterized by transient-FFT (NOT `.noise`)** ‚ÄĒ **WORKS (0.00 dB).** Split detected lines into fundamentals vs IM/harmonic products (the linear model emits only fundamentals; the user's nonlinearity regenerates IM). PSS/HB manifest: GCD-fold commensurate tones vs declare separate incommensurate fundamentals. External supply/bias spurs are NOT emitted ‚ÄĒ they ride the PSRR path.

## Load dependence

- **Parameter scheduling: each param a clamped quadratic in ln(iload)** ‚ÄĒ **WORKS as the default; FAILS at the edge.** The 3-corner, 0-residual-DOF quadratic is the PROVEN OVERFIT LOCUS (in-corner error ‚Č° 0, carries no generalization info; held-out 20‚Äď30√ó; flat/frozen outside [20¬Ķ,250¬Ķ] via clamp). Band-aids (_pexpr clamp, MIN_LOG_GAP, branch-B 40% gate) censor the symptom, not the correctness ‚ÄĒ they are local heuristics, not AIC/BIC.

## Large-signal / recovery (the WuR PLL thread)

In the order it evolved ‚ÄĒ most are RETIRED; read top-to-bottom to see why:
- **Hand-tuned `la_override` + Lreg‚ÄĖRreg recovery network (La=120 ¬ĶH)** ‚ÄĒ **REJECTED** (a crutch; a real user can't know La/Lreg; must be fitted). AC-inconsistent (La120 |Z|@10MHz=558 ő© vs measured ~160).
- **"Drop slew, go pure LTI with the deployment decap"** ‚ÄĒ **SUPERSEDED/RETRACTED.**
- **Fit La + recovery from AC Zout alone** ‚ÄĒ **SUPERSEDED** (a single inductor can't make the rising shelf; AC gives 24 ¬ĶH, the transient seemed to want ~120 ¬ĶH = the small-signal-vs-large-signal gap).
- **slew + recovery network (compensated.va) under decap** ‚ÄĒ **REJECTED** (crashes to the 300 mV anti-windup clamp, RMS 92‚Äď105 mV).
- **Add a branch-A slew term for the startup undershoot** ‚ÄĒ **REJECTED** (2nd expert panel, unanimous: WRONG SIGN ‚ÄĒ the coverage dip is minimized at SRa=‚ąě; a positive SRa makes the already-over-predicted dip catastrophically worse, 245‚Üí1047‚Üí4456 mV).
- **Fit anything to real_V** ‚ÄĒ **REJECTED** (overfit 2.47‚Üí35.5 mV held-out; real_V is a turn-on/cold-start ENVELOPE, corr(I,V)‚Čą0, NOT a load transient ‚Üí reference-only, never a gate).
- **Higher-order ladder Zout** (above) ‚ÄĒ **WORKS** ‚ÄĒ is the load-transient fix from the standard flow.
- **UNLOAD-overshoot: branch-A DISCHARGE (Route 1)** ‚ÄĒ **WORKS / SHIPPED (2026-07-01).** The fit-inductor L_a stores ¬ĹL¬∑I¬≤ of the DC load current, so a hard UNLOAD kicks the rail up (behavioral tank, lossless). The lesson first: **an OUTPUT-side clamp cannot fix it** (see ¬ßRefuted) ‚ÄĒ because dI_L/dt = V_across_L/L_a, clamping the overshoot small ‚áí small inductor voltage ‚áí slow drain, so overshoot-depth √ó recovery-time is ~constant. Route 1 instead DRAINS the inductor: a series term in the RESISTOR-form branch-A regulation `V(regnode,vrg) <+ R_a¬∑I + gate(vov)¬∑srcblk(I)¬∑ovVmax¬∑tanh((ovR/ovVmax)¬∑I)`, a **voltage-BOUNDED reverse EMF** applied to the whole series inductor chain that forces dI_L/dt strongly negative INDEPENDENTLY of the (bounded) overshoot ‚Üí both axes beat the clamp. Three properties make it safe, each load-bearing: (1) **gate = one-sided deadzone in the overshoot** `(vov>ovVdz)?tanh((vov‚ąíovVdz)¬≤/ovVsc¬≤):0`, vov=V(o,vrg): ZERO value AND slope at the OP (which sits at vov<0 for every load) ‚Üí Zout/PSRR/noise/DC BIT-identical (Spectre 0.0000 %), and one-sided ‚áí the loading dip is untouched; (2) **srcblk = `0.5(1‚ąítanh(I/ovIsc))`** ~1 when branch A SOURCES (the unload energy dump), ~0 when it SINKS a sustained current ‚áí a +40 mA floating-rail abuse-sink never engages it; (3) **the reverse EMF is clamped to ¬ĪovVmax** ‚áí the regulation ALWAYS keeps conductance ~1/R_a, so the DC pin can NEVER be de-pinned ‚ÄĒ the crucial contrast with the reverted "DC current compliance" (a zero-conductance current clamp that de-pinned ‚Üí FF runaway; see ¬ßPVT). Defaults ovR=4 kő©, ovVmax=2 V, ovVsc=8 mV, ovIsc=1 mA; **`ovVdz` is now AUTO-DERIVED per-rail** (`harness/fit_ovvdz.derive_ovvdz`, folded into `fit_multiport`, PURE-PYTHON, NO sim ‚ÄĒ mirrors `derive_iassist`): `ovVdz = clamp(2.5¬∑max_f|Zout|¬∑di_ripple [+ opt-in spur/PSRR], [5 mV, 0.5¬∑(supply‚ąívreg)])`, `di_ripple = 0.10¬∑I_op` (the coverage periodic-ripple di, NOT the big assist step). Reproduces the former hardcoded 25 mV as the WuR special case (PLL 24.6 mV = 2.5¬∑50 ¬ĶA¬∑197 ő©, VCO 26.1 mV = 2.5¬∑200 ¬ĶA¬∑52 ő©); `emit_pmu_model._OVD_VDZ`=25 mV is now only the FALLBACK when a rail carries no characterized Zout (legacy/single-OP), and a manual `vfit['unload_discharge']['ovVdz']` still wins. This generalizes the floor to a DIFFERENT LDO (bigger ripple / smaller headroom / different L) instead of hard-coding this chip's value. **`ovVdz` is a LARGE-SIGNAL TRANSPARENCY FLOOR, not a DC-droop margin** (an adversarial-review finding, local-Spectre-confirmed): the dump engages for ANY excursion above vreg+ovVdz while branch A sources ‚ÄĒ which includes the positive half of a large-signal PERIODIC ripple, not just the one-time unload. Since the model's PRIMARY product is the spur/PSRR/Zout SPECTRA, ovVdz must sit ABOVE the largest legitimate periodic rail excursion or the dump clips those peaks and distorts the spectra (Spectre: ovVdz=3 mV ‚Üí **16 mV clip-distortion** on a 20 mV-above ripple; ovVdz=25 mV = 2.5√ó the ~10 mV documented char ‚Üí **0.002 mV**, transparent). Result (local Spectre, deployed params, 4 mA‚Üí0.3 mA/1 ns/20 pF): **PLL +302 mV(>supply)/~2 ¬Ķs ‚Üí +34 mV(‚Č§supply), monotone settle (no 4 ¬Ķs hang); VCO +124 mV ‚Üí +28.5 mV**, no chatter; HB+PSS converge; survives minimal; +40 mA pinned at 0.824 V. TRADE-OFF: ovVdz is also where the overshoot SETTLES, so sub-¬Ķs-to-5mV recovery (which needs ovVdz<5 mV) was traded for the spectral transparency of the model's primary function ‚ÄĒ the fix still kills the two real pathologies (above-supply kick + 4 ¬Ķs hang). BAKED localparams (INTERNAL, default-ON on the resistor regulation; slew rails keep slew()+Imax); per-rail `vfit['unload_discharge']` disables/retunes (lower ovVdz IFF the rail's large-signal ripple is known < that floor). `emit_pmu_model._ovd_*`; `harness/test_unload_discharge_core.py` (incl. a ripple-transparency guard). Numbers: DATA ¬ß5.
