==================== 本回合要干的（按优先级）====================

[1] 二进制 PSF 读取器（最后一块，最重要）
    问题：ade Maestro 写的是 BINARY PSF；cadence/psf.py 只会读 ASCII（spectre_cli 用
    -format psfascii）。所以 ade 数据出来了但 importmp 读不进去（UnicodeDecodeError）。
    试过且无效：asiSetSimOptionVal 'rawfmt / asiSetEnvOptionVal 'rawFmt 等都被拒（None）。
    方案（推荐 A）：写一个独立的 BINPSF 读取器（保持 npz firewall 不依赖实时会话；集群周一
    出的也是二进制）。验证法：把 ade 生成的 input.scs 用 spectre -64 -format psfascii 再跑一遍
    （复用 cadence/spectre_run.py 思路）得到 ASCII 参照，断言「二进制读取器(ade的ac.ac) ==
    ASCII读取器(同一netlist的ascii)」。真实 ac.ac 头 16 字节：
    00 00 02 00 00 00 00 15 00 00 02 f4 00 00 00 21 ...（big-endian、分段 HEADER/TYPE/SWEEP/
    TRACE/VALUE/TOC，可参考 libpsf 格式）。读出来要喂进 cadence/insitu/importmp.py 的 read 路径。

[2] #4 GUI 卡死 / 无进度 / 不能取消 / 不能重跑
    现状：run_ade 的 poll 循环每组等到 [0,0] 最长 600s，GUI 的 _ExtractWorker 只在最末尾发
    done/failed，中途无进度；ade run 若慢/不完成就「假死」。要：run_ade 加进度回调 + 取消检查；
    GUI worker 流式显示「第 N/8 组 running…」、加 Cancel、加重跑、合理超时。
    注意：1610/1707 修好后模态框不再弹，所以「卡死」主要是 poll 无反馈 + 无取消。
